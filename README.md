# NexusOCR — מנוע OCR מחקרי לכתבי רש"י וספרות תורנית

זיהוי מתקדם של כתבי יד ודפוס תורני באמצעות בינה מלאכותית, ברמה מחקרית.

המערכת מותאמת במיוחד עבור **כתב רש"י**, **דפוס ישן**, **צילום לא איכותי** וסריקות מורכבות,
כאשר היעד המרכזי הוא **מינימום דילוגי מילים ומינימום הזיות מודל — גם במחיר זמן עיבוד ארוך יותר**.

---

## ✨ עיקרי השדרוג (Research Grade Engine)

### 1. זיהוי אוטומטי של סוג הכתב (Classification)
לפני הפענוח מתבצע שלב סיווג שמזהה את סוג הכתב, ומפעיל פרומפט OCR ייעודי לכל סוג:
- כתב רש"י
- דפוס רגיל
- שילוב רש"י ודפוס
- כתב יד
- טקסט מעורב

בנוסף מזוהה **סוג המסמך התורני** (בבלי, ירושלמי, רמב"ם, טור, בית יוסף, שו"ע, משנה ברורה, שו"ת, ראשונים, אחרונים ועוד) לצורך אופטימיזציית פריסה — **מבלי** להשתמש בידע זה להשלמת תוכן.

### 2. מצב OCR ייעודי לכתב רש"י
כאשר מזוהה כתב רש"י המנוע נכנס למצב חומרה מרבית:
- קריאה אות-אחר-אות, ללא ניחוש מילים.
- ללא השלמות אוטומטיות, ללא תיקון כתיב, ללא היגיון תחבירי.
- ללא השלמות תלמודיות, ללא המרת מילים למונחים מוכרים.
- מילה לא ברורה מסומנת ב-`-------` ולא מנוחשת.

### 3. Multi-Pass OCR אמיתי — 7 מעברים (במקום 4)
כל מעבר הוא קריאת API עצמאית מול המודל:

| מעבר | תיאור |
|------|-------|
| Pass 1 | קריאה גולמית |
| Pass 2 | קריאה שנייה עצמאית (על וריאנט תמונה נפרד) |
| Pass 3 | השוואת Pass 1 ו-Pass 2 והכרעה לפי התמונה |
| Pass 4 | חיפוש מילים/אותיות/שורות חסרות |
| Pass 5 | בדיקת אותיות דומות (ב/כ, ד/ר, ה/ח, ו/ז, י/ו, ן/ו) |
| Pass 6 | בדיקת ראשי תיבות (ללא הרחבה/פענוח) |
| Pass 7 | אימות סופי מול התמונה + ציוני אמינות |

### 4. מצב Ultra Rashi — 10 מעברים
מצב להשגת דיוק מקסימלי (עדיפות לדיוק על פני מהירות):
- 10 מעברי OCR (כולל בדיקת ניקוד/פיסוק/מבנה, בדיקת שלמות, וקריאה עצמאית שלישית).
- כל שלבי העיבוד המקדים פעילים, רזולוציה מקסימלית.
- אפס השלמות מהזיכרון.

### 5. OCR לפי אזורים (Layout Segmentation)
פירוק העמוד לבלוקים (כותרת, גוף, רש"י, הערות שוליים, ציטוטים) באמצעות ניתוח פריסה,
פענוח כל בלוק בנפרד, ואיחוד למסמך מסודר. מומלץ לעמודים מורכבים.

### 6. Preprocessing מתקדם (בדפדפן)
לפני שליחת התמונה למודל:
- **Super Resolution** — הגדלת רזולוציה + חידוד.
- **Deskew** — יישור דפים עקומים.
- **Contrast Enhancement** — שיפור ניגודיות (percentile stretch + גמא).
- **Adaptive Threshold** — הבלטת אותיות חלשות (וריאנט בינארי לקריאה עצמאית).
- **Sharpen** — חידוד טקסט (unsharp mask).
- **Denoise** — ניקוי רעשים (median filter; פעיל אוטומטית ב-Ultra).

### 7. רזולוציה גבוהה
חילוץ PDF משודרג מ-`scale 1.5` ל-`scale 3.0` (ועד `5.0` ב-Ultra / לפי בחירה),
עם הגנת זיכרון אוטומטית, ורוחב OCR מוגדל מ-`1200px` ל-`2048px` (ועד `3000px+`).

### 8. מנגנון Anti-Hallucination
אסור למודל להשלים פסוקים, גמרא, ביטויים מוכרים, ראשי תיבות או ציטוטים מהזיכרון.
כל קטע לא ברור מסומן ב-`-------`.

### 9. Confidence Score לכל מילה
המעבר הסופי מחזיר ציון אמינות (0–100) לכל מילה. מילים מתחת ל-**85** מסומנות
חזותית (הדגשה צהובה) ונספרות לבדיקה. הציונים נכללים בייצוא JSON.

### 10. ייצוא מתקדם
- **DOCX אמיתי** (ספריית `docx`, לא MHTML) — RTL מלא, פונט עברי, מעברי עמוד, ותמונות מוטמעות אופציונלית.
- **PDF** — ייצוא איכותי (RTL, פונט עברי) דרך חלון הדפסה.
- **TXT** — טקסט בלבד (עם BOM לקידוד עברית תקין).
- **JSON** — טקסט + ציוני אמינות לכל מילה.

> תאימות לאחור: אם ספריית `docx` נכשלת בטעינה, הייצוא נופל אוטומטית ל-MHTML (כמו בגרסה הקודמת).

## 🛠️ מבנה הפרויקט
פרויקט **Vite + React + TypeScript + Tailwind CSS** מוכן לפריסה:

```
├─ index.html              # נקודת כניסה ל-Vite
├─ package.json            # תלויות וסקריפטים
├─ vite.config.ts          # תצורת Vite
├─ tailwind.config.js      # תצורת Tailwind
├─ postcss.config.js
├─ tsconfig.json
├─ vercel.json             # תצורת פריסה ל-Vercel
└─ src/
   ├─ main.tsx             # bootstrap של React
   ├─ App.tsx              # מנוע ה-OCR והממשק (הקובץ המרכזי)
   ├─ index.css            # Tailwind
   └─ vite-env.d.ts
```

> התלויות הכבדות (`pdf.js`, `docx`) נטענות מ-CDN בזמן ריצה — אין צורך להתקין אותן.

## 🚀 הרצה מקומית
```bash
npm install
npm run dev      # שרת פיתוח (http://localhost:5173)
npm run build    # בנייה לפרודקשן → dist/
npm run preview  # תצוגה מקדימה של ה-build
```

## ☁️ פריסה ל-Vercel
1. חברו את המאגר ל-Vercel — הוא יזהה אוטומטית את framework **Vite** (build: `npm run build`, output: `dist`).
2. הגדירו משתנה סביבה **`VITE_GEMINI_API_KEY`** עם מפתח Gemini (Settings → Environment Variables), או הזינו מפתח ישירות בממשק (⚙️ הגדרות) — הוא נשמר מקומית בדפדפן.
3. Deploy.

> **חשוב:** ללא מפתח Gemini תקין הסריקה לא תפעל. השיגו מפתח חינמי ב-[Google AI Studio](https://aistudio.google.com/apikey).

## 📖 שימוש
1. לחצו "הוסף קבצים" (תמונות JPG/PNG או PDF).
2. (אופציונלי) פתחו ⚙️ הגדרות לבחירת מצב Ultra Rashi / OCR לפי אזורים / סוג כתב / רזולוציה / שלבי עיבוד, והזנת מפתח API.
3. לחצו "התחל סריקה רב-שלבית".
4. ייצאו לפורמט הרצוי דרך תפריט "ייצוא".
